# MagicSquare_1004 — Mom Test 질문 10개

**페르소나:** 4×4 격자, 빈칸 2개(0), 1~16, 합 34 맞추는 학습자

---

## 1. 마지막으로 틀렸을 때

### Q1 ✅
**마지막으로 4×4 마방진에서 빈칸 2개를 채웠는데 합이 안 맞았던 때, 그 순간 격자에서 먼저 확인한 줄·열·대각선이 뭐였어?**

> 과거 구체 사실, 솔루션 없음. “틀렸을 때 실제로 뭘 봤는지” 행동을 끌어냄.

### Q2 ❌
**자동으로 행·열·대각선 합을 검증해 주는 도구가 있으면, 틀린 걸 바로 잡을 수 있을 것 같아?**

- **왜 나쁜가:** 가상의 미래·솔루션 찬성 여부를 묻는 질문. “좋을 것 같다”는 답은 검증 가치가 없음.
- **✅ 고친 버전:** **그때 틀렸다는 걸 알게 된 계기가 뭐였어? 누가/뭐 때문에 그걸 확인하게 됐어?**

---

## 2. 검증/확인 비용

### Q3 ✅
**가장 최근에 합=34인지 확인할 때, 행·열·대각선을 전부 더하는 데 대략 몇 분 걸렸어? 그중 가장 오래 걸린 부분은 뭐였어?**

> 시간·비용이라는 측정 가능한 과거 사실. “검증에 드는 노력”을 숫자로 끌어냄.

### Q4 ❌
**`SquareValidator` 같은 검증 모듈을 만들면 확인 시간이 줄어들지 않을까?**

- **왜 나쁜가:** 클래스·모듈이라는 구현 솔루션을 전제로 의견을 묻음. Mom Test는 “이미 한 검증”이지 “만들면 어떨까”가 아님.
- **✅ 고친 버전:** **마지막으로 합을 확인할 때, 손으로 다 더했어 아니면 어딘가에 적어 가며 했어? 그 방법을 택한 이유가 뭐였어?**

---

## 3. 포기·중단 순간

### Q5 ✅
**가장 최근에 4×4 부분 마방진(빈칸 2개) 풀다가 중간에 그만둔 적 있어? 그때 몇 분째였고, 뭐 때문에 멈췄어?**

> 중단 시점·이유라는 과거 사실. “포기”의 구체적 트리거를 파악.

### Q6 ❌
**단계별 힌트가 있으면 포기하지 않고 끝까지 풀 수 있을 것 같아?**

- **왜 나쁜가:** 힌트 기능이라는 솔루션에 대한 희망·의견만 듣게 됨. 실제로 포기한 경험을 검증하지 못함.
- **✅ 고친 버전:** **막혀서 손을 뗀 그날, 그다음에 뭘 했어? 다시 격자를 봤어, 아니면 완전히 다른 일로 넘어갔어?**

---

## 4. 같은 문제를 다시 풀 때

### Q7 ✅
**전에 틀렸거나 막혔던 같은 4×4 퍼즐을 나중에 다시 풀었을 때, 처음이랑 비교해서 뭐가 달랐어? (시간, 확인 순서, 실수 여부 등)**

> 재시도 경험의 변화를 과거 사실로 비교. 학습·습관 변화를 드러냄.

### Q8 ❌
**이전에 풀던 문제를 저장해 두는 앱이 있으면, 다시 풀기가 훨씬 편할 거야?**

- **왜 나쁜가:** 저장 앱이라는 제품 가정 + “편할 거야”라는 미래 예측. 실제 재풀이 행동을 묻지 않음.
- **✅ 고친 버전:** **같은 문제를 두 번째로 풀 때, 첫 번째 때 썼던 숫자나 메모를 다시 봤어? 안 봤다면 그냥 처음부터 다시 시작했어?**

---

## 5. 다른 사람에게 설명/공유할 때

### Q9 ✅
**마지막으로 누군가에게 “빈칸 2개, 합 34” 마방진 푸는 법을 설명할 때, 상대가 어디서 막혔고 그때 너는 뭘 보여 줬어?**

> 설명·공유의 실제 장면. “누가·어디서·어떻게” 막혔는지가 진짜 문제 단서.

### Q10 ❌
**설명용 튜토리얼이나 영상을 만들면, 다른 사람한테 훨씬 잘 가르칠 수 있지 않을까?**

- **왜 나쁜가:** 콘텐츠 제작 솔루션에 대한 찬성을 유도. 실제로 설명하다 막힌 경험을 검증하지 않음.
- **✅ 고친 버전:** **설명하다가 “이건 말로는 안 되겠다” 싶어서 포기했던 적 있어? 그때 어떤 단계(빈칸 찾기, 숫자 배치, 합 확인)였어?**
