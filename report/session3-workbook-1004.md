## 1) 주제 (한 문장)

> **4×4 부분 마방진에서 행·열·대각선 합(34)을 빠짐없이 판정하고, 틀린 축을 바로 알 수 있는지 검증한다.**

- Mom Test 축: **판정 불확실 → 비용(20분) → 암산 재검증**에 대응
- 솔루션 최소화: “앱/자동완성 프로그램”이 아니라 **판정 가능 여부**가 주제

---

## 2) R-G-I-O

| | 내용 |
|---|------|
| **Role** | **검증자(Validator)** — 학습자 대신 “맞다/틀리다”를 말해 주지 않고, **어떤 축이 34인지·아닌지**만 판정하는 역할 |
| **Goal** | 빈칸 2개를 채운 4×4 격자에 대해 **행 4 + 열 4 + 대각선 2 = 10개 축**을 누락 없이 검사하고, 실패 시 **어느 축인지** 식별한다 |
| **Input** | `int[4][4]` 격자 (0 = 빈칸, 1~16 = 배치된 값), 목표 합 `34` |
| **Output** | `ValidationResult`: 전체 통과 여부 + 실패 축 목록 (예: `ROW_2`, `DIAG_MAIN`) + (선택) 축별 합 값 |

### 예시 Input (STEP 1 퍼즐 — 빈칸 채운 후)

```
 16 |  3 |  2 | 13
  5 | 10 | 11 |  8   ← (2,3) 채움
  9 |  6 |  7 | 12   ← (3,2) 채움
  4 | 15 | 14 |  1
```

## 3) 성공 기준 3개 (Mom Test 증거 연결)

| # | 성공 기준 | 연결된 Mom Test 증거 |
|---|-----------|----------------------|
| **S1** | **10개 축**(행 4·열 4·대각선 2)을 한 번의 호출로 모두 검사하며, 대각선 2개를 테스트 케이스에 **반드시** 포함한다 | ① 「**대각선 하나를 빼먹어서** 20분 날렸다」— 대각선 누락이 핵심 실패 |
| **S2** | 검증 결과가 **암산 없이** 즉시 나온다 (통과/실패 + 실패 축 이름). 학습자가 “맞는 것 같다”에서 멈추지 않게 **판정 근거**를 반환한다 | ② 「**중간에 암산으로 검증해봤는데** 그때 알게 됐어」— 수동 검증 의존 제거 |
| **S3** | 의도적으로 **한 축만 틀린** 격자를 넣으면, **20분 뒤가 아니라 1초 안에** 어느 축이 34가 아닌지 특정한다 (Test Loop로 재현) | ① 20분 낭비 + ③ ECB 과제에서 **실패 조건 식별** 요구와 연결 |

## 4) 표면 문제 — 이번 프로젝트에서 **하지 않을 것**

> Mom Test **표면 문제**를 그대로 스코프 밖으로 둔다.

| 하지 않음 | 이유 (Mom Test) |
|-----------|-----------------|
| **4×4 부분 마방진 자동 검증 “완성 앱”**을 만든다 | 솔루션이 이미 정의됨 — 진짜 문제는 **판정 확신**이지 제품 형태가 아님 |
| **빈칸 자동 채우기 / Solver 전체** | 증거는 “검증 누락”이지 “풀이 알고리즘 부재”가 아님 |
| **GridUI · InputHandler · ResultDisplay** (Boundary 전체) | 세션 3 범위 밖; UI 없이 Command·Test로 판정만 검증 |
| **MissingFinder · ECB 전 계층 완성** | Entity/Control/Boundary 통합은 후속 세션 |
| **“TDD 하면 좋겠다” 수준의 방법론 논의** | Mom Test 금지 질문류 — **Test Loop는 S3 재현 수단**으로만 사용 |

**한 줄 요약:** 이번 세션은 **풀기·앱 만들기**가 아니라, **10개 축 판정을 Test Loop로 고정**하는 것까지.

## 5) 8계층 — 이번 세션에서 만드는 것

| 계층 | 이번 세션 | 산출물 |
|------|-----------|--------|
| **Rule** | ✅ 만듦 | 마방진 합 규칙: 행/열/대각선 = 34, 1~16 중복 없음, 빈칸(0) 허용 여부 정의, 실패 조건 ≥3 (합 불일치·중복·범위 밖·빈칸 잔존 등) |
| **Command** | ✅ 만듦 | `validate(grid)` 또는 CLI `validate --file grid.txt` — Input 격자 → Output `ValidationResult` |
| **(Skill)** | ⭕ 선택 | `check_all_sums(grid, target=34)` — Rule을 호출하는 재사용 단위; Command가 Skill을 감싸도 됨 |
| **Test Loop** | ✅ 만듦 | Red → Green → Refactor: ① 정답 격자 통과 ② 대각선만 틀린 격자 실패(S1) ③ 행만 맞고 열 틀린 격자(S3) |
| Entity | ❌ 안 만듦 | `MagicSquare`, `Cell` — 후속 |
| Control | ❌ 안 만듦 | `Solver`, `MissingFinder` — 후속 |
| Boundary | ❌ 안 만듦 | `GridUI`, `ResultDisplay` — 후속 |
| Integration | ❌ 안 만듦 | E2E·제출 파이프라인 — 후속 |

### 계층별 최소 체크리스트

**Rule**
- [ ] `TARGET_SUM = 34`
- [ ] 검사 축 10개 명시 (행4·열4·대각2)
- [ ] 실패 조건 3가지 이상 문서화 (Mom Test S3·ECB 과제 연계)

**Command**
- [ ] 격자 입력 → `ValidationResult` 출력
- [ ] 실패 시 축 이름 반환 (암산 대체)

**(Skill)**
- [ ] (선택) Rule 순수 함수로 분리해 Command·Test에서 공유

**Test Loop**
- [ ] S1: 대각선 누락 버그가 다시 나오지 않는 테스트
- [ ] S2: 한 번 실행으로 전 축 결과 확인
- [ ] S3: 틀린 축 특정 테스트 (1초 이내는 로컬 단위 테스트로 대체)
