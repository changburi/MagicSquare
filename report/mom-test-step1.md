## 1. 페르소나

### 프로젝트 ECB 스케치 (작업 당시 기준)

**Entity**
- `MagicSquare` — 4×4 격자 데이터
- `Cell` — 값, 위치, 상태
- `SolveResult` — 풀이 결과

**Control**
- `SquareValidator` — 합 유효성 검증
- `MissingFinder` — 빈칸 위치 탐색
- `Solver` — 완성 알고리즘

**Boundary**
- `GridUI` — 격자 시각화
- `InputHandler` — 사용자 입력
- `ResultDisplay` — 결과 표시

## 3. 표면 문제 (잘못된 정의 — 솔루션 섞임)

> 「4×4 부분 마방진 **자동 검증 프로그램**을 만들어서 행·열·대각선 합을 한 번에 확인한다」

**왜 잘못됐나:** 제품·기능이 이미 답에 포함되어 있음. Mom Test에서는 **과거 행동**으로 검증해야 하며, 인터뷰 대상이 원하는 **솔루션**이 아니라 **이미 겪은 불편**을 뽑아야 함.

---

## 4. 진짜 문제 (한 문장)

> 부분 마방진을 채운 뒤 **모든 합 조건(특히 대각선)을 빠짐없이 검증했는지 스스로 확신하기 어렵고**, 틀린 줄 모르고 진행하다 **암산으로 다시 확인할 때까지 수십 분의 작업 비용을 날린다.**

| 축 | 내용 |
|----|------|
| 불편 | 대각선 등 일부 조건을 놓친 채 “맞다”고 착각 |
| 비용 | 잘못된 가정 하에 **약 20분** 소요 |
| 판정 | “다 맞췄다”는 **자기 판정이 틀릴 수 있음** |
| 재현 | 중간 **암산 검증**을 해야만 오류 발견 |

---

## 5. Mom Test 증거 (인용 3줄)

1. 「빈칸 2개 넣고 행·열·대각선 합 맞췄는데 **대각선 하나를 빼먹어서 20분 날렸다**」
2. 「**중간에 암산으로 검증해봤는데** 그때 알게 됐어」
3. 「(첨부한 것과 같이) **ECB 개념 분석·4×4 마방진** 작업을 했는데, 어려움이 있었어」

## 6. 다음 단계 (STEP 2 힌트)

- **미완 응답 보완:** 암산 검증 시 어떤 축(행/열/대각선)을 실제로 확인했는지 기록
- **실패 조건 연결:** “대각선 누락”을 ECB의 `SquareValidator` 책임 범위와 매핑
- **검증 가능한 가설:** “합 검증 체크리스트 없이 손으로 풀면, 대각선 조건을 빼먹을 확률이 높다” — 추가 인터뷰 또는 자기 관찰로 확인
